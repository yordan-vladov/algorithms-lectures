# Стек и Опашка

---

## Какво ще научим днес?

- Какво е Стек (Stack) и принципът LIFO
- Какво е Опашка (Queue) и принципът FIFO
- Примерна имплементация на C++
- Основни операции и тяхната сложност
- `std::stack` и `std::queue` - стандартните класове
- Кога да използваме Стек срещу Опашка

## Мотивация

- ArrayList и LinkedList позволяват достъп до всеки елемент
- Понякога ни трябва **ограничен достъп** - само от единия край
- Два основни модела:
  - **Стек** - последният влязъл, първи излиза
  - **Опашка** - първият влязъл, първи излиза

```
Реален свят:
  Стек  → купчина чинии (взимаш отгоре)
  Опашка → опашка на каса (първи дошъл, първи обслужен)
```

---

## Стек (Stack) - LIFO

- **LIFO** = Last In, First Out
- Добавяме и премахваме само от **върха** (top)

```
push(1)   push(2)   push(3)   pop()

  [ ]       [ ]      [3] ← top   [ ]
  [ ]      [2] ← top [2]        [2] ← top
  [1] ← top [1]      [1]        [1]
```

- Само операции върху върха → елементите под него са недостъпни

## Опашка (Queue) - FIFO

- **FIFO** = First In, First Out
- Добавяме от **задния край** (back), премахваме от **предния край** (front)

```
enqueue(1)  enqueue(2)  enqueue(3)  dequeue()

front        front        front        front
  [1]          [1][2]      [1][2][3]    [2][3]
                                ↑ back       ↑ back
```

- Елементите се обработват в реда, в който са постъпили

## Основни операции на Стек

| Операция  | Описание                             |
| --------- | ------------------------------------ |
| `push(x)` | Добавя елемент на върха              |
| `pop()`   | Премахва елемента от върха           |
| `top()`   | Връща елемента от върха (без премахване) |
| `empty()` | Проверка дали стекът е празен        |
| `size()`  | Брой елементи                        |

---

## Основни операции на Опашка

| Операция    | Описание                                      |
| ----------- | --------------------------------------------- |
| `enqueue(x)`| Добавя елемент в задния край                  |
| `dequeue()` | Премахва елемента от предния край             |
| `front()`   | Връща предния елемент (без премахване)        |
| `back()`    | Връща задния елемент (без премахване)         |
| `empty()`   | Проверка дали опашката е празна               |
| `size()`    | Брой елементи                                 |

## Имплементация на Стек - структура

```cpp
#include <iostream>
#include <vector>
using namespace std;

class Stack {
private:
    vector<int> data;

public:
    void push(int x) {
        data.push_back(x);
    }

    void pop() {
        if (!empty()) data.pop_back();
    }

    int top() const {
        return data.back();
    }

    bool empty() const {
        return data.empty();
    }

    int size() const {
        return data.size();
    }
};
```

- Използваме `vector` - `push_back` / `pop_back` са O(1) амортизирано

---

## Имплементация на Стек - визуализация

```
Stack s;

s.push(10);   data: [10]
s.push(20);   data: [10, 20]
s.push(30);   data: [10, 20, 30]

s.top()  → 30
s.pop()  →   data: [10, 20]
s.top()  → 20
s.size() → 2
```

## Имплементация на Опашка - структура

```cpp
#include <iostream>
#include <vector>
using namespace std;

class Queue {
private:
    vector<int> data;
    int head;   // индекс на предния елемент
    int tail;   // индекс след последния елемент
    int count;

    void resize() {
        vector<int> newData(data.size() * 2);
        for (int i = 0; i < count; i++)
            newData[i] = data[(head + i) % data.size()];
        data = move(newData);
        head = 0;
        tail = count;
    }

public:
    Queue(int cap = 16)
        : data(cap), head(0), tail(0), count(0) {}
```

---

## Имплементация на Опашка - кръгов буфер

```cpp
    void enqueue(int x) {
        if (count == (int)data.size()) resize();
        data[tail] = x;
        tail = (tail + 1) % data.size();  // кръгово
        count++;
    }

    int dequeue() {
        int val = data[head];
        head = (head + 1) % data.size();  // кръгово
        count--;
        return val;
    }

    int front() const { return data[head]; }
    bool empty()  const { return count == 0; }
    int  size()   const { return count; }
};
```

- **Кръгов буфер** - избягваме местенето на елементи, O(1) за enqueue/dequeue
- Използваме `vector` за storage - няма нужда от ръчно `new`/`delete`

---

## Кръгов буфер - визуализация

```
capacity = 5

enqueue(A): [ A , _ , _ , _ , _ ]   head=0, tail=1
enqueue(B): [ A , B , _ , _ , _ ]   head=0, tail=2
enqueue(C): [ A , B , C , _ , _ ]   head=0, tail=2
dequeue():  [ _ , B , C , _ , _ ]   head=1, tail=3  → A
enqueue(D): [ _ , B , C , D , _ ]   head=1, tail=4
enqueue(E): [ _ , B , C , D , E ]   head=1, tail=0  (кръгово!)
```

---

## Разширяване на буфера при пълен капацитет

```
capacity = 4,  head=2, tail=2  (пълен: B, C, D, E)

[ D , E , B , C ]
        ↑ head=2

resize() → разгъва в нов масив с capacity=8:

[ B , C , D , E , _ , _ , _ , _ ]
  ↑ head=0              ↑ tail=4
```

- При `enqueue` когато `count == capacity`: удвояваме размера
- Елементите се **разгъват** в ред от `head` — кръговото наместване изчезва
- `head = 0`, `tail = count` след resize
- Сложност: O(1) амортизирано (като `vector::push_back`)

## Пример за употреба - Стек

```cpp
int main() {
    Stack s;

    s.push(1);
    s.push(2);
    s.push(3);

    while (!s.empty()) {
        cout << s.top() << " ";  // 3 2 1 (обратен ред)
        s.pop();
    }
}
```

**Приложение:** Обръщане на последователност - елементите излизат в обратен ред

---

## Пример за употреба - Опашка

```cpp
int main() {
    Queue q(10);

    q.enqueue(1);
    q.enqueue(2);
    q.enqueue(3);

    while (!q.empty()) {
        cout << q.front() << " ";  // 1 2 3 (същия ред)
        q.dequeue();
    }
}
```

**Приложение:** Обработка на задачи по реда на постъпване

## Сложност на операциите

| Операция         | Стек  | Опашка | Бележка                          |
| ---------------- | ----- | ------ | -------------------------------- |
| push / enqueue   | O(1)* | O(1)*  | *амортизирано - удвояване при пълен буфер |
| pop / dequeue    | O(1)  | O(1)   | кръгов буфер при опашката                |
| top / front      | O(1)  | O(1)   | само четене                      |
| Памет            | O(n)  | O(n)   |                                  |

- Всички ключови операции са **O(1)** - оптимални структури за съответния достъп

## `std::stack` - стандартната имплементация

- C++ предоставя `std::stack` в `<stack>`
- По подразбиране използва `std::deque` отдолу

```cpp
#include <iostream>
#include <stack>
using namespace std;

int main() {
    stack<int> s;

    s.push(10);
    s.push(20);
    s.push(30);

    cout << s.top() << endl;  // 30
    s.pop();
    cout << s.top() << endl;  // 20
    cout << s.size() << endl; // 2
}
```

---

## Основни методи на `std::stack`

```cpp
stack<int> s;

s.push(x);       // добавя x на върха
s.pop();         // премахва върха (не връща стойност!)
int v = s.top(); // връща стойността на върха
bool b = s.empty();  // true ако е празен
int n = s.size();    // брой елементи
```

> **Внимание:** `pop()` не връща стойността - трябва да вземете `top()` преди `pop()`!

```cpp
// Правилно:
int val = s.top();
s.pop();

// Грешно - стойността е загубена:
s.pop();
```

## `std::queue` - стандартната имплементация

- C++ предоставя `std::queue` в `<queue>`

```cpp
#include <iostream>
#include <queue>
using namespace std;

int main() {
    queue<int> q;

    q.push(1);   // enqueue
    q.push(2);
    q.push(3);

    cout << q.front() << endl;  // 1
    q.pop();                    // dequeue
    cout << q.front() << endl;  // 2
    cout << q.size()  << endl;  // 2
}
```

---

## Основни методи на `std::queue`

```cpp
queue<int> q;

q.push(x);         // добавя x в задния край
q.pop();           // премахва предния елемент
int f = q.front(); // стойността на предния елемент
int b = q.back();  // стойността на задния елемент
bool e = q.empty();
int  n = q.size();
```

## Реални приложения на Стек

- **Извикване на функции** - call stack (рекурсия)
- **Undo/Redo** в редактори
- **Проверка на скоби** - `(()[])` валидно ли е?
- **Обръщане на низ**
- **Оценяване на изрази** - `3 + 4 * 2`

```
Проверка на скоби: ( [ ) ]
  push '('  → стек: [ ( ]
  push '['  → стек: [ (, [ ]
  pop за ')' → очакваме '[', намерихме '(' → НЕВАЛИДНО!
```

## Реални приложения на Опашка

- **Планировчик на процеси** (OS scheduler)
- **Принтерна опашка** - документи се печатат по ред
- **BFS** (Breadth-First Search) - обхождане на граф
- **Буфер за мрежови пакети**

```
BFS от връх A:
  queue: [A]
  обработи A → добави съседи B, C  →  queue: [B, C]
  обработи B → добави съсед D      →  queue: [C, D]
  обработи C →                     →  queue: [D]
  ...
```

## Стек срещу Опашка

| Характеристика    | Стек (Stack)          | Опашка (Queue)         |
| ----------------- | --------------------- | ---------------------- |
| Принцип           | LIFO                  | FIFO                   |
| Добавяне          | На върха              | В задния край          |
| Премахване        | От върха              | От предния край        |
| Реален аналог     | Купчина чинии         | Опашка на каса         |
| Типична употреба  | Рекурсия, Undo, скоби | BFS, задачи, буфери    |

## Кога да използваме Стек?

**Стек** - когато:
- Обработката трябва да е в **обратен ред** на постъпването
- Имаме **вложена** структура (рекурсия, скоби, дървета)
- Трябва да "помним" предишно състояние и да се върнем към него

**Примери:** проверка на скоби, DFS, undo операции, изчисляване на изрази

---

## Кога да използваме Опашка?

**Опашка** - когато:
- Обработката трябва да е в **същия ред** на постъпването
- Имаме **ниво по ниво** обхождане (BFS)
- Моделираме чакаща редица (принтер, scheduler)

**Примери:** BFS в граф, принтерна опашка, буфер за съобщения

## Обобщение

- **Стек** реализира LIFO - добавяме и премахваме само от върха
- **Опашка** реализира FIFO - добавяме отзад, премахваме отпред
- Всички ключови операции са **O(1)**
- Имплементация: `vector` за стек, **кръгов буфер** за опашка
- `std::stack` и `std::queue` са готовите имплементации в C++
- Изборът зависи от реда на обработка: обратен (стек) или същи (опашка)
