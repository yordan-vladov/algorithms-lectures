## Мотивация

- Как бихте изчислили факториел на число?
- **Пример:**
```
5! = 5 × 4 × 3 × 2 × 1
```

---
- Обикновено използваме цикъл.

```cpp
#include <iostream>
using namespace std;

int factorial(int n) {
    int result = 1;

    for (int i = 1; i <= n; i++) {
        result *= i;
    }

    return result;
}

int main() {
    int num = 5;
    cout << "Factorial: " << factorial(num) << endl;
    return 0;
}
```

---

- Има и друг начин. Фунцкията може да извиква себе си за реши предишните стъпки на задачата:
```cpp
#include <iostream>
using namespace std;

int factorial(int n) {
    if (n == 0 || n == 1)
        return 1;

    return n * factorial(n - 1);
}

int main() {
    cout << factorial(5);
}
```

## Какво е рекурсия?

- Рекурсия е техника, при която функция извиква сама себе си, за да реши по-малка версия на същия проблем.
- Основна идея:
    - Големият проблем → по-малък проблем → още по-малък → решение
---

## Основни компоненти на рекурсията

- Всяка рекурсивна функция има:
    1. Базов случай (условие за спиране)
    2. Рекурсивна стъпка (извикване на функцията)

- Без базов случай → безкрайна рекурсия → срив на програмата

## Пример: Факториел

- Математически:
    - `n! = n × (n − 1)!`
- Базов случай:
    - `0! = 1`

---

## Факториел - C++ код

```cpp
#include <iostream>
using namespace std;

int factorial(int n) {
    if (n == 0 || n == 1)
        return 1;

    return n * factorial(n - 1);
}

int main() {
    cout << factorial(5);
}
```

---

## Как работи?

- `factorial(5)`
    - → `5 * factorial(4)`
    - → `5 * 4 * factorial(3)`
    - → `5 * 4 * 3 * factorial(2)`
    - → `5 * 4 * 3 * 2 * factorial(1)`
    - → `5 * 4 * 3 * 2 * 1`
---

## Пример: Сума от 1 до n

```cpp
int sum(int n) {
    if (n == 1)
        return 1;

    return n + sum(n - 1);
}
```

---

## Пример: Fibonacci

```cpp
int fib(int n) {
    if (n <= 1)
        return n;

    return fib(n - 1) + fib(n - 2);
}
```

**Проблем:**
- Много повтарящи се изчисления → бавен алгоритъм

---

## Кога използваме рекурсия?

- Рекурсията е подходяща когато:
    * Проблемът може да се раздели на подпроблеми
    * Структурата е рекурсивна по природа
    * Решението се описва математически рекурсивно

---

- **Примери:**
    * Дървета
    * Графи
    * Сортиране (Merge Sort, Quick Sort)
    * Файлова система
---

## Как работи вътрешно?

- Използва се структура:
    - Call Stack (стек на извикванията)
- **Пример:**
```
factorial(3)

Stack:

factorial(3)
factorial(2)
factorial(1)
```
- След това започва връщане назад.

---

## Предимства на рекурсията

* По-кратък код
* По-четим алгоритъм
* Естествено решение за дървета и графи
* По-лесно мислене при някои задачи

---

## Недостатъци на рекурсията

* Използва повече памет
* По-бавна в много случаи
* Риск от stack overflow
* По-трудна за дебъгване

## Рекурсия срещу Итерация

Итеративен факториел:

```cpp
int factorial_iter(int n) {
    int result = 1;

    for (int i = 1; i <= n; i++) {
        result *= i;
    }

    return result;
}
```
